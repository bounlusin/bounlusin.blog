markdown学习总结

Markdown是什么

Markdown 是一种轻量级标记语言。它允许人们“使用易读易写的纯文本格式编写文档，然后转化成有效的XHTML(或HTML)文档”。

详情参见：wiki-Markdown

为什么使用Markdown

  • 纯文本，兼容性极强，可以用任何文本编辑器打开
  • 让使用者更专注于文字内容本身而不是排版
  • 格式转换方便，Markdown的文本可以轻松转换为html、电子书等
  • Markdown的标记语法与几号的可读性

Markdown语法

标题

在#  #之间填写标题内容

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

注：# 和「一级标题」之间建议保留一个字符的空格，这是最标准的 Markdown 写法。

列表

无序列表

- 文本1
- 文本2
- 文本3

有序列表

1. 文本1
2. 文本2
3. 文本3

注：-、1.和文本之间要保留一个字符的空格。

链接和图片

插入链接

[显示文本](链接地址)    
例如
[bounlusin的博客](http://bounlusin.github.io)

插入图片

![](图片链接地址)    
例如
![](http://7xivjb.com1.z0.glb.clouddn.com/luffy.jpg)

引用

在引用内容前面加上 > 即可
例如 > 发现更大的世界

注：> 和文本之间要保留一个字符的空格。

粗体和斜体

用两个*包含的文本即粗体
用一个*包含的文本即斜体

以上便是一些比较常用的Markdown语法。

我的第一篇文章

这是我自己用hexo搭建blog之后写的第一篇文章。

从25号开始到现在，从什么都不会到成功，心里的满足感不言而喻。

这期间接触到的一些东西：

  • git

  • ruby

  • DevKit

  • jekyll

  • node.js

  • hexo

  • markdown

期间遇到不少困难，大部分还是吃了英文不好的亏。

原本走的是 

git-->ruby-->devkit-->jekyll

这条路，但是jekyll安装过程中有个不明原因的错误，至今尚未查明。

偶然发现了 hexo，于是转投

git-->node.js-->hexo

效果不错。

2015/4/28 0:48:20

update：

其实昨晚并没有完全调试好，hexo生成的静态网页在我电脑上可以访问，却没有成功推送到github上，原因是

hexo deployer

这个指令总是失败。

查了资料发现hexo安装时忘了执行下面这个指令

npm install hexo-deployer-git --save

更正之后即可成功推送到github上，且不用

git push origin master

2015/4/28 17:31:53

不管怎么样，这个blog算是搭建起来了，以后我会在这里写一些学习总结，知识管理以及个人感悟。

为什么要写blog呢？

我觉得重要的不是一定要写出来什么东西给别人看，重要的是写blog过程中的自我思考和进步，当然写出来好的东西分享给大家也是好的。

有很多时候我们都不知所措，茫然，不知道该做什么，该怎么做，做的是否正确。自我思考对于我们来说，很重要。保持热情和毅力去做事固然重要，但停下来思考也是必不可少的。

写blog的目的就在于此：自我思考。
